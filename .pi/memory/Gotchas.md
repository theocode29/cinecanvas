- **2026-07-02** — Spec §4.1 assumed the overlay titlebar strip stays natively draggable with no custom drag region. This is FALSE in Tauri v2 on macOS: titleBarStyle Overlay + hiddenTitle produces an immovable window (tauri#9503, regression from v1). → **In Tauri v2 overlay/frameless windows, ALWAYS add an HTML element with `data-tauri-drag-region` (and grant `core:window:allow-start-dragging` in capabilities). CineCanvas uses a transparent 32px absolute drag overlay above the full-height canvas so no top band is visible.**

- **2026-07-02** — Using string `"free"` for `panOnScrollMode` prop on ReactFlow — @xyflow/react v12 uses `PanOnScrollMode` enum, not a string union. → **Always import `PanOnScrollMode` and use `{PanOnScrollMode.Free}` (or `.{Vert}`) on the ReactFlow component.**

- **2026-07-02** — Using `CanvasImageData` directly as `NodeProps<CanvasImageData>` or `useNodesState<CanvasImageData>` — @xyflow/react v12 `NodeData` constraint requires `Record<string, unknown>` index signature, which a clean interface fails. → **Define a local intersection type `type MyNodeData = CanvasImageData & Record<string, unknown>` in the canvas module only, keeping the public types.ts pure. Use `Node<MyNodeData, 'image'>` for NodeProps and useNodesState.**

- **2026-07-02** — Applying `filter: url(#cc-grain)` to a full-window overlay above React Flow — this forces expensive repaint/compositing work while panning and makes the empty canvas feel laggy. → **Do not put SVG/CSS filters or blend-mode overlays across the moving canvas viewport.
