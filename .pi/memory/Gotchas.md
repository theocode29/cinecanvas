- **2026-07-02** — Running `pnpm tauri dev` inside a subagent/non-GUI bash context — it blocks waiting for a window and compiles Rust for minutes, hanging the run. → **Never run `pnpm tauri dev`/`cargo build`/`cargo check` inside subagents. Verify Tauri config via `pnpm vitest run` + `pnpm tsc --noEmit` + reading the config files; launch `tauri dev` only manually from a GUI session.**

- **2026-07-02** — Running `pnpm create tauri-app . -f` in a non-empty directory — the `-f/--force` flag wiped non-template files, deleting `cinecanvas-spec.md` (the source of truth) and `.pi-subagents/` (which crashed the worker runner with ENOENT when it tried to write its output). AGENTS.md and .pi/ survived but unreliably. → **Never run `create-tauri-app . -f` in a directory containing protected files. Scaffold into a fresh empty subdir, then move only the template files into the project root (preserving AGENTS.md/spec/.pi). After scaffolding, verify protected files survive before any other step.**

- **2026-07-02** — Spec §4.1 assumed the overlay titlebar strip stays natively draggable with no custom drag region. This is FALSE in Tauri v2 on macOS: titleBarStyle Overlay + hiddenTitle produces an immovable window (tauri#9503, regression from v1). → **In Tauri v2 overlay/frameless windows, ALWAYS add an HTML element with `data-tauri-drag-region` (and grant `core:window:allow-start-dragging` in capabilities). CineCanvas uses a 32px parchment strip at the top of App.tsx; the canvas sits in a flex:1 area below it.**

- **2026-07-02** — Using string `"free"` for `panOnScrollMode` prop on ReactFlow — @xyflow/react v12 uses `PanOnScrollMode` enum, not a string union. → **Always import `PanOnScrollMode` and use `{PanOnScrollMode.Free}` (or `.{Vert}`) on the ReactFlow component.**

- **2026-07-02** — Using `CanvasImageData` directly as `NodeProps<CanvasImageData>` or `useNodesState<CanvasImageData>` — @xyflow/react v12 `NodeData` constraint requires `Record<string, unknown>` index signature, which a clean interface fails. → **Define a local intersection type `type MyNodeData = CanvasImageData & Record<string, unknown>` in the canvas module only, keeping the public types.ts pure. Use `Node<MyNodeData, 'image'>` for NodeProps and useNodesState.**
